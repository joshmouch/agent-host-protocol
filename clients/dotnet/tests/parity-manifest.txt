# .NET parity manifest — the expected test methods, in executable form.
#
# This is the machine-checked form of the cross-language test master matrix in
# OpenAgency plan docs/plans/proposed/2026-06-04-0137-ahp-dotnet-client-test-parity.
# The parity gate (scripts/check-test-parity.sh) greps each MethodName across
# clients/dotnet/tests/; any that are absent are reported as "missing", grouped
# by phase, so a failing gate names exactly which parity tests still need writing.
#
# Format:  MethodName | TargetSuite | phase | matrix-row
#   - lines starting with '#' and blank lines are ignored
#   - phase 1 = test-only gaps (the feature already exists) — the "+45"
#   - phase 2 = feature+test gaps (new production code, then its tests)
#
# Existing tests (the baseline 23) are NOT listed here — they are already present;
# this file enumerates only the parity tests still to be ADDED. Deletion of an
# existing test is caught separately by the count floor (tests/MIN_TEST_COUNT).
